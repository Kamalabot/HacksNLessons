There are so many libraries in many languages today, for visualising data.

The libraries and the visualisation objective are deciding which language you can use.

  • Python has Plotly, Altair
  • R has Shiny
  • JS has D3, Vega, Plot, Vega lite, and PlotlyJS
  • There is mother of all visualisation libraries, GGplot

Grammar of Graphics[GG] is a way of creating the charts. Among the above the **best and easily accessible** is Observable Plot. A go to tool for visualisation. The **best and toughest **to master is D3 but gives maximum flexibility.

Charts are user facing products. So the color, scale, the type of plot (or in case of observable the Marks) used determine how the data is percieved.

Allowing the user to explore the data and find the insights by themselves is another very interesting objective.

Think about the trade off between the frameworks based on the application and decide appropriately. I will go with D3 as the best, followed by Plote In python Altair and Plotly. In R, shiny.
